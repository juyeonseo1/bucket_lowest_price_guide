업무 개요
목적: 오늘의집(버킷플레이스) 타겟 상품의 시장 내 최저가 정보를 정확히 확인·검증하는 작업입니다. 각 플랫폼의 동일 옵션 상품 가격을 비교하여 시스템 수집 데이터의 정확성을 판단하고, 필요한 경우 직접 최저가를 산출합니다.
SCOPE 작업 범위
    - 매일 '작업 대상'에 업데이트 되는 내용하고 본인의 범위 확인 후 작업
RULE 판단 기준 요약
  • 네이버와 타 스토어 가격 동일 → 네이버를 최저가로 선택
  • 가격 다를 경우 → 적립금 제외 실 판매가 기준 최저가 선택
  • 네이버 적립금은 최저가 판단 기준에서 제외
  • 네이버 포인트는 로그인 시 확인 (포인트 포함 시)

최저가 선정 기준
  • 네이버와 타 스토어 가격이 동일한 경우 → 네이버를 최저가로 선택
  • 가격이 다른 경우 → 적립금을 제외한 실 판매가 기준으로 가장 낮은 곳 선택
  • 네이버 적립금은 최저가 판단 기준에서 제외
  • 네이버 포인트는 로그인해서 확인 (포인트 포함 일 시)
쿠폰 적용 제외 기준
아래 항목은 최저가 판단 시 쿠폰 적용 제외 대상입니다.

  • 네이버 라운지 전용 쿠폰
  • 카드 할인
  • 첫구매 쿠폰
  • 플랫폼별로 다양한 쿠폰이 있으므로 더블체크 필요
  • 쿠폰이 반영된 실 판매가가 기재된 경우: 그 금액으로 최저가 기록
동일 상품 판별 기준
  • 상품명 + 수량 + 용량이 동일한 상품만 동일 상품으로 간주
  • 혼합 세트 구성품이 포함된 경우 → 동일 상품 제외
  • 핵심 키워드 누락 시 제외
  • 수량/용량 불일치 시 제외
플랫폼별 수집 기준

- 오늘의 집 <- > 네이버 가격비교에서 검색하여 비교

- 네이버 가격비교에 나오는 실제 판매 플랫폼의 정보를 가져온다. ㄷㄷㄷㄷㄷ

수집 제외 대상
작업 흐름 (Step-by-Step)
1

타겟 상품 확인

당일 배정된 버킷플레이스 타겟 URL/상품 목록 확인. 행 기준으로 작업 수량이 결정됩니다 

2

수집 개수 확인

후보 리스트 수집 개수를 확인합니다. 0개이면 레포트 작성, 5개 미만이면 네이버 가격비교에서 브랜드+상품명으로 검색, 5개 이상이면 후보 리스트에서 최저가 검증 진행.

3

동일 상품 판별

상품명 + 수량 + 용량 기준으로 동일 상품 여부를 판별합니다. 혼합 세트 포함, 핵심 키워드 누락, 수량/용량 불일치 시 제외 처리.

4

최저가 산출

동일 상품으로 판별된 후보들에서 적립금/쿠폰 제외 실 판매가 기준 최저가를 선택. 네이버와 타 스토어가 동일 가격이면 네이버 선택.

5

결과 입력 및 레포트


6

DS 관리자 최종 교차 검수

자주 하는 질문
적립금과 포인트를 최저가에 포함해야 하나요?▾
네이버 적립금은 최저가 판단 기준에서 제외합니다. 네이버 포인트는 로그인해서 확인하고, 포인트 포함 일 시 반영하여 최저가를 판단합니다. 
수집 후보가 0개인 경우 어떻게 처리하나요?▾
수집 후보가 0으로 나온 것은 아직 수집중인 데이터로, 대기 후 수집이 보일 시 작업하시면 됩니다. 
네이버와 타 스토어 가격이 같을 때 어떤 기준으로 선택하나요?▾
네이버를 최저가로 선택합니다. 가격이 다른 경우에는 적립금을 제외한 실 판매가 기준으로 가장 낮은 곳을 선택합니다.ㄱㄱㄱㄱㄱㄱ
7% 가구 브랜드데이 할인 쿠폰은 최저가 적용 가능한가요?▾
플랫폼별, 상품별로 다양한 쿠폰이 있으므로 더블체크가 필요합니다. 고객과 합의한 기준은: 네이버 라운지 전용 쿠폰, 카드할인, 첫구매 쿠폰은 제외합니다. 쿠폰이 반영된 실 판매가가 기재되어 있다면 그 금액으로 기록하는 것이 맞습니다. 애매한 특이사항은 슬랙에 공유해주세요.
애매한 케이스 처리 기준은 무엇인가요?▾
판단 기준이 불명확한 경우, 슬랙 채널에 케이스를 공유하여 가이드 업데이트를 요청합니다. 관리자에게 처리 방법을 문의합니다. 
유통기한 임박 상품의 매칭은 어떻게 하나요?▾
동일하게 유통기한 임박 상품에 대해서만 매칭합니다.
2순위 작업은 어떻게 하나요?▾
- 오늘의 집이 최저가인 경우, 두번째로 싼 가격의 상품 정보를 기재합니다.
ㄴ 오늘의 집 제외한 매칭 url이 하나라도 있을 경우 적용
 ㄴ 리툴 수집 후보군에는 매칭 url이 없더라도, 추가 네이버 검색시 추가 매칭 url을 발견했던 경우도 적용한다. 
가격 적용의 기준은 어떻게 되나요?▾
url을 열어 최종 가격으로 수집합니다.

매칭 플랫폼에 Unknown 이라고 적혀져있어요. / 이상한 url이 적혀져 있어요.▾
매칭 플랫폼이 Unknown 이고 url 이 이상한 경우 : 시스템에서 오류로 찾은 행이므로 수집후보 리스트에서 무시하셔도 됩니다. 해당 행을 무시하라는 것이고, 최저가 찾는 작업은 그대로 진행하셔야 합니다. 예를 들어 Unknown으로 되어 있는 거 하나만 있으면, '시스템이 잘 못 찾았다'라고 판단하고 네이버 가격비교에 직접 검색하셔서 찾으셔야 합니다.
