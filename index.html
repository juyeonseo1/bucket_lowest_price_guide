업무 개요
목적: 오늘의집(버킷플레이스) 타겟 상품의 시장 내 최저가 정보를 정확히 확인·검증하는 작업입니다. 각 플랫폼의 동일 옵션 상품 가격을 비교하여 시스템 수집 데이터의 정확성을 판단하고, 필요한 경우 직접 최저가를 산출합니다.
SCOPE 작업 범위
  • 네이버 / 다나와 / 에누리 / 지마켓 카탈로그에 있는 상품
  • 쿠팡, 11번가 등 주요 플랫폼의 동일 상품 후보
  • DB에 없는 경우 레포트 작성 (수집 가능 여부 확인)
  • 최저가 판단 기준은 적립금 제외한 실 판매가 기준
RULE 판단 기준 요약
  • 네이버와 타 스토어 가격 동일 → 네이버를 최저가로 선택
  • 가격 다를 경우 → 적립금 제외 실 판매가 기준 최저가 선택
  • 네이버 적립금은 최저가 판단 기준에서 제외
  • 네이버 포인트는 로그인 시 확인 (포인트 포함 시)
  • 쿠팡: 로켓상품 우선, 없으면 최저가 일반상품

  • 쿠팡: 로켓상품 우선, 없을 시 최저가 일반상품

동일 상품 판별 기준
  • 상품명 + 수량 + 용량이 동일한 상품만 동일 상품으로 간주
  • 혼합 세트 구성품이 포함된 경우 → 동일 상품 제외
  • 핵심 키워드 누락 시 제외
  • 수량/용량 불일치 시 제외
  • 고양이모래 무향 ≠ 향 정보가 다른 경우 → no_match 처리가 더 안전
플랫폼별 수집 기준

🟢 네이버

  • 최저가 상품 (1 page, 40개 기준 검색)
  • 검색 결과 전체 PDP 들어가서 수집
  • 해당 내용 LLM 매칭 후 화면 노출
  • 로그인 상태에서 포인트까지 확인
  • 낮은가격순 + 적용기준 off = 잘 나옴

🟡 쿠팡

  • 로켓상품 우선 선택
  • 로켓상품 없을 시 최저가 일반상품
  • 쿠팡 target URL 기준 수집
  • HTTP 직접 요청 403 차단 → browser 기반 수집
  • 로켓배송 여부 별도 확인 필요

🔵 다나와 / 에누리

  • 최저가 상품 (1 page, 40개 기준 검색)
  • 카탈로그 기반 동일 상품 판별
  • 네이버 가격비교 URL 적합성 확인 필요

🟠 G마켓 / 11번가

  • 최저가 상품 (1 page, 40개 기준 검색)
  • 삼성에서 제공한 검색어로 조회 후 매칭 상품 찾기
  • 카드 할인 정보 수집 시 별도 파서 개발 필요

작업 흐름 (Step-by-Step)
1

타겟 상품 확인

당일 배정된 버킷플레이스 타겟 URL/상품 목록 확인. 행 기준으로 작업 수량이 결정됩니다 (1회차 1,300~1,700개 처리).

2

수집 개수 확인

후보 리스트 수집 개수를 확인합니다. 0개이면 레포트 작성, 1개이면 네이버 가격비교에서 브랜드+상품명으로 검색, 1개 이상이면 후보 리스트에서 최저가 검증 진행.

3

동일 상품 판별

상품명 + 수량 + 용량 기준으로 동일 상품 여부를 판별합니다. 혼합 세트 포함, 핵심 키워드 누락, 수량/용량 불일치 시 제외 처리.

4

최저가 산출

동일 상품으로 판별된 후보들에서 적립금/쿠폰 제외 실 판매가 기준 최저가를 선택. 네이버와 타 스토어가 동일 가격이면 네이버 선택.

5

결과 입력 및 레포트

어드민 작업 툴에 결과 입력. DB에 없는 수집 불가 상품은 레포트 작성하여 DS 관리자에게 전달. 애매한 케이스는 슬랙에 남겨 확인 요청.

6

DS 관리자 최종 교차 검수

DS 관리자가 최종 교차 검수 후 버킷플레이스 측에 전달. All Pass 건에 대해서만 최종 교차 검수 진행 (6:30분경 확인).

자주 하는 질문
적립금과 포인트를 최저가에 포함해야 하나요? ▾
네이버 적립금은 최저가 판단 기준에서 제외합니다. 네이버 포인트는 로그인해서 확인하고, 포인트 포함 일 시 반영하여 최저가를 판단합니다. 쿠팡의 경우 쿠폰이 반영된 실 판매가가 기재된 경우 그 금액으로 기록합니다.
수집 후보가 0개인 경우 어떻게 처리하나요? ▾
DB에 없는 경우 레포트를 작성합니다. 이후 PoC 기간에 검색 최적화를 미리 해두면 고도화가 가능합니다. 다나와, 에누리, 네이버 카탈로그에 있는 것들에 대해서라도 수집을 시도하고, DB에 없으면 레포트 작성합니다.
네이버와 타 스토어 가격이 같을 때 어떤 기준으로 선택하나요? ▾
네이버를 최저가로 선택합니다. 가격이 다른 경우에는 적립금을 제외한 실 판매가 기준으로 가장 낮은 곳을 선택합니다.
쿠팡에서 수집이 403 오류로 막히면 어떻게 하나요? ▾
쿠팡은 단순 HTTP requests 방식은 403으로 차단됩니다. browser 기반 수집으로 전환하거나, enhans-scraper 경유 방식을 사용합니다. 직접 요청이 막혔을 때는 브라우저 자동화 기반 수집이 현실적인 대안입니다.
코스트코 상품은 어떻게 처리하나요? ▾
코스트코는 옵션이 있으면 일단 전부 제외입니다. 지마켓/옥션의 경우 옵션 선택 시 별도 상품코드로 빠지는 경우만 가능합니다. 옵션이 없는 코스트코 단품은 수집 가능합니다.
7% 가구 브랜드데이 할인 쿠폰은 최저가 적용 가능한가요? ▾
플랫폼별, 상품별로 다양한 쿠폰이 있으므로 더블체크가 필요합니다. 고객과 합의한 기준은: 네이버 라운지 전용 쿠폰, 카드할인, 첫구매 쿠폰은 제외합니다. 쿠폰이 반영된 실 판매가가 기재되어 있다면 그 금액으로 기록하는 것이 맞습니다. 애매한 특이사항은 슬랙에 공유해주세요.
식품류 수집 제외 기준은 무엇인가요? ▾
기본적으로 김치, 샐러드 등 신선 식품류는 제외합니다. 단, 가공식품인 경우는 괜찮습니다 (예: 해물 누룽지탕 냉동팩). 동일 상품 판별이 어렵고 유통 특성상 가격 변동이 심한 신선 식품은 제외하는 것이 원칙입니다.
수집 배치는 어떤 일정으로 운영하나요? ▾
현재는 매일 오전 1시~9시 배치 형태로 운영됩니다. 중장기적으로는 24시간 상시 동작 형태 전환을 검토 중이며, 이 경우 시간당 수집 모수가 낮아져 더 많은 상품 정보를 효율적으로 수집할 수 있습니다. 전체 상품을 일주일에 나누어 수집하는 방식도 검토 중입니다.
애매한 케이스 (pending) 처리 기준은 무엇인가요? ▾
판단 기준이 불명확한 경우 pending으로 처리하고, 슬랙 채널에 케이스를 공유하여 가이드 업데이트를 요청합니다. 예시: 에너지효율등급 등급 판단 기준이 애매한 것들, 청소년보호/성인인증상품의 일반 기준을 벗어나는 상품들은 발견 후 검수 대상으로 추가하는 식으로 사후 처리 가능하지만 사전에 완벽하게 차단하기 어렵습니다.
